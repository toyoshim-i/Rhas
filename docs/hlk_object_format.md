# HLK オブジェクトファイルフォーマット

## 概要

X68000のリンカHLK（High-speed LinKer）が扱うオブジェクトファイル（`.o`）のフォーマットです。
HASアセンブラが生成するオブジェクトファイルもこのフォーマットに準拠します。

ファイルは**レコード**の連続で構成され、各レコードは2バイトのレコードコードで始まります。
全データはビッグエンディアンです。

## ファイル全体構造

```
[ファイルヘッダ]
  $D000 レコード（ファイルサイズ + ソース名）

[セクションヘッダ（各セクション分）]
  $C0xx レコード（セクションサイズ + セクション名）×N

[オプション情報]
  $E001 レコード（requestファイル名）  ← ある場合のみ
  $B204 レコード（アラインメント情報）  ← -gオプション等の場合のみ

[外部シンボル情報]
  $B2xx レコード（シンボル値 + シンボル名）×N

[オブジェクトコード本体]
  各種コードレコード...

[終端]
  $0000
```

## レコード形式

### $D000 — ファイルヘッダ

```
オフセット  サイズ  内容
$00         2       $D0, $00（レコードコード）
$02         4       ファイル本体サイズ（このフィールドの直後から$0000終端の直後まで）
$06         可変    ソースファイル名（拡張子なし、null終端）
```

**注意**: ファイル本体サイズはパス3の最後にファイルを書き戻して書き込まれます（前方参照）。

### $C0xx — セクションヘッダ

```
オフセット  サイズ  内容
$00         2       $C0, xx（xxはセクション番号）
$02         4       セクションサイズ（バイト数）
$06         可変    セクション名（null終端）
```

セクション番号（xx）とセクション名：
| xx | セクション名 | 説明 |
|---|---|---|
| $01 | `text` | コード（テキスト）セクション |
| $02 | `data` | 初期化済みデータセクション |
| $03 | `bss` | 未初期化データセクション |
| $04 | `stack` | スタックセクション |
| $05 | `rdata` | 相対データセクション（64KB以内） |
| $06 | `rbss` | 相対BSSセクション（64KB以内） |
| $07 | `rstack` | 相対スタックセクション（64KB以内） |
| $08 | `rldata` | 相対データセクション（64KB以上） |
| $09 | `rlbss` | 相対BSSセクション（64KB以上） |
| $0A | `rlstack` | 相対スタックセクション（64KB以上） |

**デフォルト**: 相対セクションを使用しない場合は $01〜$04 の4セクション。
相対セクションを使用する場合は $01〜$0A の10セクション。

### $E001 — requestファイル

```
オフセット  サイズ  内容
$00         2       $E0, $01
$02         可変    requestファイル名（null終端）
```

### $B204 — アラインメント情報

`-g`（SCDデバッグ）オプション使用時、または `.align` 疑似命令使用時に出力されます。

```
オフセット  サイズ  内容
$00         2       $B2, $04
$02         4       最大アライン値（2^n の n）
$06         可変    '*' + ソースファイル名 + '*'（null終端）
```

### $B2xx — 外部シンボル情報

```
オフセット  サイズ  内容
$00         2       $B2, xx（xxはシンボル種別）
$02         4       シンボル値 / シンボル番号 / コモンエリアサイズ
$06         可変    シンボル名（null終端）
```

シンボル種別（xx）：
| xx | 定数 | 意味 | 値フィールドの意味 |
|---|---|---|---|
| $FA | `SECT_GLOBL` | グローバル（外部参照/定義） | シンボル値（定義済み）またはシンボル番号（未定義） |
| $FB | `SECT_XDEF` | 外部定義（.xdef） | シンボル値 |
| $FC | `SECT_RLCOMM` | コモンエリア（64KB以上相対） | コモンエリアサイズ |
| $FD | `SECT_RCOMM` | コモンエリア（64KB以内相対） | コモンエリアサイズ |
| $FE | `SECT_COMM` | コモンエリア | コモンエリアサイズ |
| $FF | `SECT_XREF` | 外部参照 | シンボル番号（1から連番） |

## オブジェクトコード本体

ヘッダおよびシンボル情報の後に続くオブジェクトコード部分は、HAS内部では「テンポラリファイルの中間コードを処理してオブジェクトファイルに出力する」形で生成されます。

### 実際にオブジェクトファイルに書き出される内容

1. **機械語バイト列** — `out1bobj`（1バイト）、`out1wobj`（1ワード）等で出力
2. **外部参照修正情報** — オブジェクトコード内に埋め込まれた外部参照アドレスの修正用
3. **終端マーク** `$0000` — オブジェクトコード終端

### オブジェクトコードの特徴

HASが生成するオブジェクトファイルは「再配置可能形式」です。
リンカが最終アドレスを決定するまで、外部参照アドレスやセクション内オフセットは仮の値として埋められています。

## 実際のファイル例（概念）

```
D0 00                    ; $D000: ファイルヘッダ
00 00 01 2A              ; ファイル本体サイズ = 298バイト
73 6F 75 72 63 65 00     ; ソース名 "source"

C0 01                    ; $C001: textセクション
00 00 00 1E              ; サイズ = 30バイト
74 65 78 74 00           ; セクション名 "text"

C0 02                    ; $C002: dataセクション
00 00 00 0A              ; サイズ = 10バイト
64 61 74 61 00           ; セクション名 "data"

C0 03                    ; $C003: bssセクション
00 00 00 04              ; サイズ = 4バイト
62 73 73 00              ; セクション名 "bss"

C0 04                    ; $C004: stackセクション
00 00 00 00              ; サイズ = 0バイト
73 74 61 63 6B 00        ; セクション名 "stack"

B2 FF                    ; $B2FF: 外部参照シンボル
00 00 00 01              ; シンボル番号 = 1
70 72 69 6E 74 66 00     ; シンボル名 "printf"

B2 FB                    ; $B2FB: 外部定義シンボル
00 00 00 00              ; シンボル値 = 0（textセクション先頭）
6D 61 69 6E 00           ; シンボル名 "main"

... [機械語バイト列] ...

00 00                    ; $0000: 終端
```

## 注意点

### ファイル本体サイズの後付け書き込み

`$D000` レコードのサイズフィールドは、オブジェクトファイル全体の生成完了後に `fmodify` を使って書き戻されます。
そのため、ファイルに最初にゼロを書き込んでおき、最後に実際のサイズを書き込む形になります。

### 外部定義シンボルの2段階出力

`SA_NODET`（定義されたが値未確定）の外部定義シンボルは：
1. パス3の最初にとりあえず仮の値で出力（`ISXDEFMOD`フラグをセット）
2. パス3終了後の `xdefmodify` で正しい値に書き戻し

### ?? で始まるシンボルの扱い

`??` で始まるシンボルは外部定義として出力されません（`outxdef0` の処理）。
これはコンパイラが内部使用するシンボルを外部公開しないための約束事です。

## Rustでの実装方針

```rust
// オブジェクトファイルの構造体
pub struct ObjectFile {
    pub source_name: String,
    pub sections: Vec<Section>,
    pub request_files: Vec<String>,
    pub align_info: Option<AlignInfo>,
    pub symbols: Vec<ExternalSymbol>,
    pub code: Vec<u8>,  // または各セクションのコード
}

pub struct Section {
    pub id: u8,       // 1=text, 2=data, 3=bss, 4=stack, ...
    pub name: &'static str,
    pub size: u32,
}

pub struct ExternalSymbol {
    pub kind: SymbolKind,  // XDef, XRef, Comm, RComm, RLComm, Globl
    pub value: u32,
    pub name: String,
}

pub enum SymbolKind {
    Globl = 0xFA,
    XDef  = 0xFB,
    RLComm = 0xFC,
    RComm = 0xFD,
    Comm  = 0xFE,
    XRef  = 0xFF,
}
```
