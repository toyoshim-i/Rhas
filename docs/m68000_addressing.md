# M68000 実効アドレスモード

## 概要

Motorola 68000シリーズプロセッサの実効アドレス（Effective Address, EA）モードの仕様です。
HASはこれらのアドレッシングモードをサポートし、`eamode.equ` と `eamode.s` で定義・処理されています。

## アドレッシングモード一覧

### データレジスタ直接 (Dn)

```
エンコード: EAC_DN = 000 | レジスタ番号(3bit)
例: D0, D1, ..., D7
```

6ビットEAフィールド: `000 rrr`（rrrはレジスタ番号）

### アドレスレジスタ直接 (An)

```
エンコード: EAC_AN = 001 | レジスタ番号(3bit)
例: A0, A1, ..., A7, SP(=A7)
```

6ビットEAフィールド: `001 rrr`

### アドレスレジスタ間接 ((An))

```
エンコード: EAC_ADR = 010 | レジスタ番号(3bit)
例: (A0), (A1), ..., (A7)
```

6ビットEAフィールド: `010 rrr`

### ポストインクリメント付きアドレスレジスタ間接 ((An)+)

```
エンコード: EAC_INCADR = 011 | レジスタ番号(3bit)
例: (A0)+, (A1)+, ..., (A7)+
```

6ビットEAフィールド: `011 rrr`

### プリデクリメント付きアドレスレジスタ間接 (-(An))

```
エンコード: EAC_DECADR = 100 | レジスタ番号(3bit)
例: -(A0), -(A1), ..., -(A7)
```

6ビットEAフィールド: `100 rrr`

### ディスプレースメント付きアドレスレジスタ間接 ((d16,An) / d(An))

```
エンコード: EAC_DSPADR = 101 | レジスタ番号(3bit)
例: (4,A0), (-2,A1), 0(A2)
```

6ビットEAフィールド: `101 rrr` + 16ビット符号付き変位

68020以降はベースディスプレースメント（32ビット）も可能（フルフォーマット拡張ワード使用）。

### インデックス付きアドレスレジスタ間接 ((d8,An,Dn.w/.l) / d(An,Dn))

```
エンコード: EAC_IDXADR = 110 | アドレスレジスタ番号(3bit)
例: (4,A0,D1.w), (A1,A2.l), ($10,A0,D3.l*4)
```

6ビットEAフィールド: `110 rrr` + 16ビット拡張ワード（ブリーフフォーマット）

#### 拡張ワード（ブリーフフォーマット、全CPU対応）
```
ビット  意味
15-12   インデックスレジスタ番号（0-7=Dn, 8-15=An）
11      インデックスレジスタサイズ（0=.w, 1=.l）
10-9    スケールファクタ（00=*1, 01=*2, 10=*4, 11=*8）※68020以降
8       フォーマット（0=ブリーフ, 1=フル）
7-0     8ビット符号付きディスプレースメント
```

#### 拡張ワード（フルフォーマット、68020以降）

`EXW_FULL`（bit8=1）のときフルフォーマット：

```
ビット  意味
15-12   インデックスレジスタ番号
11      インデックスレジスタサイズ
10-9    スケールファクタ
8       フォーマット（必ず1）
7       ベースレジスタサプレス（EXW_BS）
6       インデックスサプレス（EXW_IS）
5-4     ベースディスプレースメントサイズ（00=リザーブ, 01=ヌル, 10=ワード, 11=ロング）
3       インデックス位置（0=プリインデックス, 1=ポストインデックス）
1-0     アウタディスプレースメントサイズ（00=ヌル, 01=ヌル, 10=ワード, 11=ロング）
```

フルフォーマットでは拡張ワードに続いてベースディスプレースメント（1または2ワード）と
アウタディスプレースメント（0または2ワード）が続きます。

### 絶対ショート (xxx.W / (abs).w)

```
エンコード: EAC_ABSW = 111 | 000
例: $1000.w, ($FF00).w
```

6ビットEAフィールド: `111 000` + 16ビット符号付きアドレス（-32768〜32767に符号拡張）

### 絶対ロング (xxx.L / (abs).l)

```
エンコード: EAC_ABSL = 111 | 001
例: $00100000.l, ($FF0000).l
```

6ビットEAフィールド: `111 001` + 32ビットアドレス

### PC相対ディスプレースメント付き間接 ((d16,PC) / d(PC))

```
エンコード: EAC_DSPPC = 111 | 010
例: (label,PC), ($10,PC)
```

6ビットEAフィールド: `111 010` + 16ビット符号付き変位（PC = 変位ワードのアドレス）

### PC相対インデックス付き間接 ((d8,PC,Dn.w/.l))

```
エンコード: EAC_IDXPC = 111 | 011
例: (label,PC,D0.w), ($10,PC,A1.l)
```

6ビットEAフィールド: `111 011` + 拡張ワード（インデックス付きアドレスレジスタ間接と同形式）

68020以降はフルフォーマット拡張ワードも使用可能。

### イミディエイト (#data)

```
エンコード: EAC_IMM = 111 | 100
例: #1, #$FF, #'A'
```

6ビットEAフィールド: `111 100` + データ（サイズにより1/2/4バイト）

## アドレッシングモードのビット定義（HAS内部）

HASは命令がどのEAモードを受け付けるかをビットマスクで管理しています：

```
EA_DN     = bit0   データレジスタ直接
EA_AN     = bit1   アドレスレジスタ直接
EA_ADR    = bit2   アドレスレジスタ間接
EA_INCADR = bit3   ポストインクリメント
EA_DECADR = bit4   プリデクリメント
EA_DSPADR = bit5   ディスプレースメント付き
EA_IDXADR = bit6   インデックス付き（ベース/メモリ間接も同じbit）
EA_ABSW   = bit7   絶対ショート
EA_ABSL   = bit8   絶対ロング
EA_DSPPC  = bit9   PC相対ディスプレースメント
EA_IDXPC  = bit10  PC相対インデックス（ベース/メモリ間接も同じbit）
EA_IMM    = bit11  イミディエイト
```

### アドレッシングモードグループ

```
EG_DATA = DN | ADR | INCADR | DECADR | DSPADR | IDXADR | ABSW | ABSL | DSPPC | IDXPC | IMM
          (アドレスレジスタ直接を除く全モード)

EG_MEM  = ADR | INCADR | DECADR | DSPADR | IDXADR | ABSW | ABSL | DSPPC | IDXPC | IMM
          (レジスタ直接を除くメモリ系モード)

EG_ALT  = DN | AN | ADR | INCADR | DECADR | DSPADR | IDXADR | ABSW | ABSL
          (可変モード = PC相対とイミディエイトを除く)

EG_CTRL = ADR | DSPADR | IDXADR | ABSW | ABSL | DSPPC | IDXPC
          (制御モード = インクリメント/デクリメントとレジスタ直接とイミディエイトを除く)
```

## サイズ指定

HASでは命令のオペレーションサイズを `.b` / `.w` / `.l` / `.s` / `.d` / `.x` / `.p` で指定します：

| サフィックス | 意味 | バイト数 |
|---|---|---|
| `.b` | バイト（8ビット） | 1 |
| `.w` | ワード（16ビット） | 2 |
| `.l` | ロングワード（32ビット） | 4 |
| `.s` | 単精度浮動小数点（32ビット） | 4 |
| `.d` | 倍精度浮動小数点（64ビット） | 8 |
| `.x` | 拡張精度浮動小数点（80ビット） | 10/12（内部は12） |
| `.p` | パック10進浮動小数点（96ビット） | 12 |

HASの内部では `SZ_*` 定数で管理（`register.equ`を参照）。

## 最適化

HASのコードオプティマイザは以下のような変換を行います：

1. **絶対ショートへの変換**: `-32768`〜`32767`の絶対アドレスを絶対ショートに
2. **PC相対への変換**: 近距離のアドレス参照をPC相対に
3. **ゼロディスプレースメントの除去**: `0(An)` → `(An)`
4. **分岐命令のサイズ最適化**: `BRA.w` → `BRA.s`（8ビット変位で届く場合）
5. **クイックイミディエイト変換**: `MOVE.l #1,Dn` → `MOVEQ #1,Dn`

## 68000 vs 68020以降の差異

| 機能 | 68000/68010 | 68020以降 |
|---|---|---|
| フルフォーマット拡張ワード | 不可 | 可 |
| ベースディスプレースメント32bit | 不可 | 可 |
| アウタディスプレースメント | 不可 | 可 |
| スケールファクタ(*2/*4/*8) | 不可 | 可 |
| PC間接32bitオフセット | 不可 | 可 |
| BRA.l | 不可 | 可（68020以降） |
| Long Bcc | 不可 | 可（68020以降） |

## HASのEAモード内部表現（EACODE）

HASは実効アドレスを解析した結果を構造体バッファ `EABUF1`/`EABUF2` に格納します：

```
オフセット  サイズ  名前          説明
$00         2       EACODE        実効アドレスモードエンコード（上記EAC_*）
$02         2       EXTCODE       拡張ワード
$04         1       EADTYPE       実効アドレスデータタイプ（EAD_*）
$05         1       EAIMMCNT      FP命令用イミディエイトデータ数
$06         1       RPNSIZE1      式のサイズ（SZ_*）
$07         1       OPTIONALPC    PCアドレッシングがオプショナルならtrue
$08         2       RPNLEN1       逆ポーランド式の長さ（-1=定数, 0=式なし）
$0A         256     RPNBUF1       逆ポーランド式バッファ（128ワード）
...         (RPNBUF2等)
```

### 実効アドレスデータタイプ（EAD_*）

| 値 | 定数 | 説明 |
|---|---|---|
| 0 | `EAD_NONEREG` | データ不要（Dn, An）|
| 1 | `EAD_NONE` | データ不要（(An), (An)+, -(An)）|
| 2 | `EAD_DSPADR` | (d,An) |
| 3 | `EAD_IDXADR` | (d,An,Rn) ブリーフ |
| 4 | `EAD_BDADR` | (bd,An,...) フル |
| 5 | `EAD_MEMADR` | ポスト/プリインデックスメモリ間接 |
| 6 | `EAD_DSPPC` | (d,PC) |
| 7 | `EAD_IDXPC` | (d,PC,Rn) ブリーフ |
| 8 | `EAD_BDPC` | (bd,PC,...) フル |
| 9 | `EAD_MEMPC` | ポスト/プリインデックスPCメモリ間接 |
| 10 | `EAD_ABSLW` | 絶対ロング/ショート |
| 11 | `EAD_IMM` | イミディエイト |
