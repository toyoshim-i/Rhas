## 完了済み項目（詳細）

### 1. 構文解析・命令サポートの欠落（全解決）
以前に ERROR と報告されていたすべての 68020/68040 拡張構文のパース・エンコード不具合が解決されました：
- **`ea_020`** (68020 EA indexed+scale 構文): **PASS** (100% 一致)
- **`insn_movem`** (単一レジスタ MOVEM パース): **PASS** (100% 一致)
- **`ea_memory_indirect`** (メモリ間接アドレッシング): **PASS** (100% 一致)
- **`insn_cache`** (CINV/CPUSH キャッシュ操作命令): **PASS** (100% 一致)
- **`insn_move16`** (MOVE16 非ポストインクリメント形式): **PASS** (100% 一致)

### 2. エンコードや生成バイナリの不一致（全解決）
以前に MISMATCH と報告されていたアセンブル出力バイナリの違いが解決されました：
- **`opt_c0` / `opt_c2`**: 最適化オプション（`-c0` での全最適化無効化、`-c2` でのv2互換モード）適用時のバイト並び不一致を完全解消。
- **`scd_g` / `scd_file`**: SCDデバッグ情報構造・レコードサイズ・ヘッダー・フッター構成を HAS060.X とバイト単位で完全一致化。
- **`insn_chk` / `insn_chk2_cmp2` / `insn_020` (EXTB等)**: 該当命令のエンコーダおよびアドレッシング符号拡張処理を修正し不一致を解消。
- **`expr_bitwise`**: ビット演算子式評価時の優先順位と結果出力を修正し完全一致。
- **`pseudo_offset` / `pseudo_rsect`**: セクション配置ルール、相対セクションの配置順とパディング定義を調整し解消。
- **`opt_moveq`**: データレジスタへの即値転送時の自動 `MOVEQ` 変換範囲と最適化ロジックの限界値をオリジナルの仕様と完全同期。

### 3. 異常系エラー検出の欠落・不整合（全解決）
以前に `#[ignore]` 指定されていた 22 件のエラーメッセージ比較用テストがすべて解消され、期待通りのエラーコードおよびエラー詳細メッセージが出力されるようになりました：
- **未定義シンボル (`test_error_undefined_symbol`)**: `ErrorCode::ExprUndef` の正確なマッピングと報告。
- **シンボル再定義 (`test_error_symbol_redefinition`)**: ラベル・定数・Predefinedシンボルの定義制限エラーマッピングの精密化。
- **CPU ゲート (`test_error_bcc_long_on_68000` / `test_error_chk_long_on_68000` 等)**: 68000/68010 モードでの Bcc.L / CHK.L の禁止検知。
- **即値・シフト・分岐範囲外 (`test_error_moveq_overflow` / `test_error_addq_overflow` 等)**: 符号付き8ビット/Addq/シフトカウントそれぞれの境界値エラーの具体的報告。
- **ゼロ除算 (`test_error_div_zero`)**: Pass 1 / Pass 3 における Rpn 式評価時の `DivisionByZero` エラーのフックと報告。
- **サイズ制限 (`test_error_address_register_byte` 等)**: アドレスレジスタへのバイトアクセス、MOVE to/from CCR/SR のサイズ制約、メモリに対するシフト・ビット操作のサイズ制約検知。
- **マクロ・制御構造の不整合 (`test_error_endm_without_macro` 等)**: 未クローズ文字列、孤立した `.endm`、`.else`、`.endif` の正確な検出。
