# HAS060X.X アーキテクチャ概要

## 概要

HAS060X.X は、X68000向けの高速アセンブラ「HAS.X v3.09」を原型とし、M.Kamadaが68060対応に改造した「HAS060.X」を、さらにTcbnErikが改造したものです。
ソースは68000アセンブリ言語で書かれており、X68000上でのみネイティブ動作します。

**Rhasプロジェクト**はこれをRustで書き直し、クロスプラットフォームで動作させることを目的としています。

## アセンブル処理フロー（3パス方式）

```
[パス1] ソース読み込み → テンポラリファイルへ中間コード出力
         ・シンボル定義の収集
         ・マクロ定義の収集
         ・命令コードサイズの仮決定（最適化可能箇所を記録）

[パス2] テンポラリファイル読み込み → テンポラリファイル更新
         ・前方参照の解決
         ・命令サイズの最適化（branch等）
         ・最適化が収束するまで繰り返す可能性あり

[パス3] テンポラリファイル読み込み → オブジェクトファイル出力
         ・最終的な機械語コードの生成
         ・外部参照・定義情報の出力
         ・シンボルテーブルの出力
```

## メモリレイアウト（X68000実機上）

```
[実行ファイル本体]
  ├── コード（HASアセンブラ自体の機械語）
  └── ワークエリア（WORKBEGIN ～ WORKBEGIN+WORKSIZE）

[動的確保メモリ]
  ├── ファイルバッファ（ソース/テンポラリ/インクルード/オブジェクト）
  ├── ラインバッファ（MAXLINELEN = 8KB）
  ├── シンボル名・マクロ登録用ワーク
  └── シンボルテーブルブロック（SYMEXTCOUNT=2000個単位で動的確保）
```

a6レジスタは常にワークエリア（WORKBEGIN）の先頭を指します。
ワークエリア内の変数は `(VARNAME, a6)` という形でアクセスされます。

## 主要定数

| 定数 | 値 | 説明 |
|---|---|---|
| `SYMHASHSIZE` | 8192 | シンボルハッシュテーブルのスロット数 |
| `CMDHASHSIZE` | 1024 | 命令名ハッシュテーブルのスロット数 |
| `INCLDMAXNEST` | 8 | インクルードファイルの最大ネスト数 |
| `MACROMAXNEST` | 32 | マクロ展開の最大ネスト数 |
| `SYMEXTCOUNT` | 2000 | 一度に確保するシンボルテーブルのエントリ数 |
| `MAXLINELEN` | 8192 | 1行の最大バイト数 |
| `LOCSYMBUFSIZE` | 16384 | マクロ内ローカルシンボル名バッファサイズ |

## シンボルテーブル構造（`symbol.equ`）

シンボルは種類によって異なる構造を持ちますが、先頭の共通部は同じです。

### 共通ヘッダ（全シンボル共通）
```
オフセット  サイズ  名前        説明
$00         4       SYM_NEXT    同一ハッシュ値の次のシンボルへのポインタ（リンクリスト）
$04         4       SYM_NAME    シンボル名へのポインタ（ローカルラベルならラベル番号）
$08         1       SYM_TYPE    シンボルの種類
$09         -       SYM_BEGIN   種類別データの開始
```

### シンボル種類（SYM_TYPE）
| 値 | 定数 | 説明 |
|---|---|---|
| 0 | `ST_VALUE` | 数値シンボル |
| 1 | `ST_LOCAL` | ローカルラベル |
| 2 | `ST_REAL` | 浮動小数点実数シンボル |
| 3 | `ST_REGSYM` | `reg`シンボル（レジスタ別名） |
| 4 | `ST_MACRO` | マクロ |
| -1 | `ST_REGISTER` | レジスタ名（組み込み） |
| -2 | `ST_OPCODE` | 命令名（組み込み） |

### 数値シンボル / ローカルラベル（ST_VALUE / ST_LOCAL）
```
オフセット  サイズ  名前          説明
$09         1       SYM_OPTCOUNT  最適化回数カウント
$0A         1       SYM_ATTRIB    定義属性（SA_UNDEF/SA_NODET/SA_DEFINE/SA_PREDEFINE）
$0B         1       SYM_EXTATR    外部定義属性（SECT_XDEF/SECT_XREF/SECT_GLOBL/SECT_COMM等）
$0C         1       SYM_SECTION   セクション番号
$0D         1       SYM_ORGNUM    orgセクション番号
$0E         1       SYM_FIRST     最初の定義方法（-1=set, 0=その他, 1=offsym）
$0F         1       (予約)
$10         4       SYM_VALUE     シンボル値
合計 $14 = 20バイト
```

### 定義属性（SYM_ATTRIB）
| 値 | 定数 | 説明 |
|---|---|---|
| 0 | `SA_UNDEF` | 参照されたが未定義 |
| 1 | `SA_NODET` | 定義されたが値未確定 |
| 2 | `SA_DEFINE` | 定義され値確定 |
| 3 | `SA_PREDEFINE` | 定義確定・再定義不可 |

### セクションコード（SYM_SECTION / SYM_EXTATR）
```
$00: SECT_ABS     定数（セクション非依存）
$01: SECT_TEXT    テキストセクション
$02: SECT_DATA    データセクション
$03: SECT_BSS     BSS（未初期化データ）セクション
$04: SECT_STACK   スタックセクション
$05: SECT_RDATA   相対データセクション（64KB以内）
$06: SECT_RBSS    相対BSS
$07: SECT_RSTACK  相対スタック
$08: SECT_RLDATA  相対データセクション（64KB以上）
$09: SECT_RLBSS   相対BSS（64KB以上）
$0A: SECT_RLSTACK 相対スタック（64KB以上）
$FA: SECT_GLOBL   グローバル（外部参照/定義）
$FB: SECT_XDEF    外部定義
$FC: SECT_RLCOMM  コモンエリア（64KB以上相対）
$FD: SECT_RCOMM   コモンエリア（64KB以内相対）
$FE: SECT_COMM    コモンエリア
$FF: SECT_XREF    外部参照
```

## テンポラリファイルの中間コード（`tmpcode.equ`）

パス1でソースを解析した結果をテンポラリファイルに書き出します。各レコードは2バイトのコードで始まります。

### 主要コード
| コード | 名前 | 説明 |
|---|---|---|
| `$0000` | `T_EOF` | ファイル終端 |
| `$0100` | `T_LINE` | 1行開始（行番号等） |
| `$0200` | `T_SYMBOL` | シンボル値定義 |
| `$0300` | `T_RPN` | 逆ポーランド式 |
| `$0400` | `T_DSPADR` | `(d,An)` アドレッシング |
| `$0500` | `T_DSPPC` | `(d,PC)` アドレッシング |
| `$0C00` | `T_BRA` | 分岐命令（bra/bsr/bcc） |
| `$1000` | `T_CONST` | アドレス非依存データ（定数） |
| `$1200` | `T_DS` | `.ds`（領域確保） |
| `$1400` | `T_ALIGN` | アライメント調整 |
| `$1500` | `T_SECT` | セクション変更 |
| `$1700` | `T_EQUCONST` | 定数の`.equ`/`.set` |
| `$1800` | `T_EQUEXPR` | 式・アドレスの`.equ`/`.set` |
| `$1900` | `T_XDEF` | `.xdef` |
| `$1E00` | `T_END` | `.end` |
| `$1F00` | `T_INCLD` | インクルードファイル開始 |
| `$2100` | `T_MACDEF` | マクロ定義開始 |

### プレデファインシンボル（`-y1`時）
- `CPU` … ターゲットCPU番号（68000, 68010, ..., 68060）
- `__DATE__` … アセンブル日付
- `__TIME__` … アセンブル時刻
- `__HAS__` … HAS バージョン番号
- `__HAS060__` … HAS060 バージョン番号
- `__HAS060X__` … HAS060X バージョン番号（-8指定時は定義されない）

## Rustへの移植方針

### モジュール構成（案）

```
src/
├── main.rs           ← エントリポイント、コマンドライン解析（clap使用）
├── context.rs        ← アセンブラ全体の状態（ワークエリアに相当）
├── symbol.rs         ← シンボルテーブル管理
├── lexer.rs          ← ソース字句解析
├── parser.rs         ← 命令解析・オペランド解析
├── expr.rs           ← 式評価
├── addressing.rs     ← 実効アドレス解析
├── pass1.rs          ← パス1（ソース読み込み→中間コード生成）
├── pass2.rs          ← パス2（最適化）
├── pass3.rs          ← パス3（オブジェクトファイル生成）
├── objfile.rs        ← HLKオブジェクトファイルフォーマット
├── instructions/
│   ├── mod.rs        ← 命令テーブル
│   ├── data_move.rs  ← データ転送命令
│   ├── integer.rs    ← 整数演算命令
│   ├── branch.rs     ← 分岐命令
│   ├── fpu.rs        ← 浮動小数点命令
│   └── ...
├── pseudo.rs         ← 疑似命令
└── macro_proc.rs     ← マクロ処理
```

### 移植時の主要課題

1. **グローバル状態の変換**: X68000版はa6レジスタがグローバル変数領域を指すが、Rustでは`Context`構造体として管理する
2. **メモリ管理**: X68000版は手動でメモリブロックを確保・管理するが、RustではVec/HashMapで自然に扱う
3. **ファイルI/O**: Human68kのDOSコールをStd I/Oに置き換え
4. **文字コード**: Shift_JIS/EUCをそのまま扱う（バイト列として）かUTF-8に変換するか検討が必要
5. **中間コード**: テンポラリファイルを実際のファイルにするかメモリ上に保持するか
6. **浮動小数点**: 68881/68882 FPU用の80bit拡張精度数値の扱い
