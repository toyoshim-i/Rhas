# Human68k システムコール仕様

## 概要

Human68kはSharp X68000のオペレーティングシステムで、Hudson Softが開発しました。
システムコールは `TRAP #1` + `.dc.w 機能番号` という形式で呼び出します。
（HASでは `DOS _FUNCNAME` というマクロで記述され、`.dc.w $FFXX` に展開されます）

### 呼び出し規約

```68000asm
; 基本形
TRAP    #1          ; $4E41
.dc.w   $FF3F       ; _READ の場合

; HASマクロを使った表記
DOS     _READ
```

引数はスタック（`-(sp)`で積む）、戻り値はD0レジスタに返ります。
エラーの場合はD0に負の値が返ります。

## HASが使用する主要システムコール

### ファイルI/O

| 番号 | 名前 | 説明 | 引数 | 戻り値 |
|---|---|---|---|---|
| `$FF3C` | `_CREATE` | ファイル作成 | `(sp)=属性, 2(sp)=ファイル名ptr` | D0=ハンドル or エラー |
| `$FF3D` | `_OPEN` | ファイルオープン | `(sp)=モード(0=r,1=w,2=rw), 2(sp)=ファイル名ptr` | D0=ハンドル or エラー |
| `$FF3E` | `_CLOSE` | ファイルクローズ | `(sp)=ハンドル(word)` | D0=エラーコード |
| `$FF3F` | `_READ` | ファイル読み込み | `(sp)=バイト数, 4(sp)=バッファptr, 8(sp)=ハンドル(word)` | D0=読み込みバイト数 or エラー |
| `$FF40` | `_WRITE` | ファイル書き込み | `(sp)=バイト数, 4(sp)=バッファptr, 8(sp)=ハンドル(word)` | D0=書き込みバイト数 or エラー |
| `$FF41` | `_DELETE` | ファイル削除 | `(sp)=リザーブ(long), 4(sp)=ファイル名ptr` | D0=エラーコード |
| `$FF42` | `_SEEK` | ファイルシーク | `(sp)=モード(0=先頭,1=現在,2=末尾), 4(sp)=オフセット, 8(sp)=ハンドル(word)` | D0=新位置 or エラー |

### 標準I/O

| 番号 | 名前 | 説明 | 引数 |
|---|---|---|---|
| `$FF09` | `_PRINT` | 文字列出力（STDOUT） | `(sp)=文字列ptr` |
| `$FF02` | `_PUTCHAR` | 1文字出力 | `(sp)=文字(word)` |

**注意**: HASではファイルハンドル1がSTDOUT、2がSTDERRです。
実際には `_WRITE` で `DOS _WRITE` + ハンドル1/2 を使って出力しています。

### メモリ管理

| 番号 | 名前 | 説明 | 引数 | 戻り値 |
|---|---|---|---|---|
| `$FF48` | `_MALLOC` | メモリ確保 | `(sp)=バイト数` | D0=確保先アドレス or エラー |
| `$FF49` | `_MFREE` | メモリ解放 | `(sp)=アドレス` | D0=エラーコード |
| `$FF4A` | `_SETBLOCK` | プロセス割り当てメモリサイズ変更 | `(sp)=サイズ, 4(sp)=開始アドレス` | D0=エラーコード |
| `$FF88` | `_MALLOC2` | メモリ確保（Human v2.15以降） | `(sp)=モード(long), 4(sp)=バイト数` | D0=確保先アドレス or エラー |
| `$FF90` | `_MALLOC3` | メモリ確保（060turbo.sys） | `(sp)=バイト数` | D0=確保先アドレス or エラー |

**HASのメモリ確保戦略**:
1. `_SETBLOCK` でプロセス自身のメモリブロックをスタック直後まで縮小
2. `_MALLOC3`（060turbo拡張）で最大メモリ確保を試みる
3. 失敗したら `_MALLOC` で最大メモリ確保

確保したメモリは以下の用途に分割：
- ソースファイルバッファ（確保量の3/8）
- テンポラリファイルバッファ（同上）
- インクルードファイルバッファ（1/8）
- オブジェクトファイルバッファ（1/8）
- 残り：シンボル名・マクロ登録用

### プロセス制御

| 番号 | 名前 | 説明 | 引数 |
|---|---|---|---|
| `$FF4C` | `_EXIT2` | プロセス終了 | `(sp)=終了コード(word)` |
| `$FF82` | `_SETENV` | 環境変数設定 | - |
| `$FF83` | `_GETENV` | 環境変数取得 | `(sp)=書き込みバッファptr, 4(sp)=変数名ptr, 8(sp)=PDB ptr(0=自分)` |

### 日時取得

| 番号 | 名前 | 説明 | 戻り値形式 |
|---|---|---|---|
| `$FF2A` | `_GETDATE` | 日付取得 | D0: `YYYYYYY.MMMM.DDDDD.WWW`（bit16-30=年,bit12-15=月,bit7-11=日,bit0-2=曜日） |
| `$FF27` | `_GETTIM2` | 時刻取得 | D0: `HHHHH.MMMMMM.SSSSS.xx`（bit27-31=時,bit21-26=分,bit16-20=秒） |

HASでは `__DATE__` と `__TIME__` プレデファインシンボルの初期化に使用。

## HASが実際に使用するシステムコール一覧

`main.s`, `file.s`, `misc.s` 等から抽出した実際の使用箇所：

```
_SETBLOCK  ... プロセスメモリブロックの縮小（meminit）
_MALLOC3   ... 060turbo拡張メモリ確保（meminit）
_MALLOC    ... 通常メモリ確保（meminit）
_OPEN      ... ソースファイルオープン（readopen）
_CLOSE     ... ファイルクローズ（各所）
_READ      ... ファイル読み込み（バッファ充填）
_CREATE    ... テンポラリ/オブジェクトファイル作成
_WRITE     ... ファイル書き込み（バッファフラッシュ）
_DELETE    ... エラー時オブジェクトファイル削除
_SEEK      ... テンポラリファイル先頭に戻る（フリーワインド）
_GETENV    ... 環境変数HAS/G2ASの取得
_GETDATE   ... アセンブル日付取得
_GETTIM2   ... アセンブル時刻取得
_EXIT2     ... アセンブル完了/エラー終了
```

## Rustへの移植時の対応

| Human68kコール | Rustの代替 |
|---|---|
| `_OPEN` / `_CLOSE` | `std::fs::File::open()` / `.close()` (Drop) |
| `_CREATE` | `std::fs::File::create()` |
| `_READ` | `Read::read()` / `BufReader` |
| `_WRITE` | `Write::write_all()` / `BufWriter` |
| `_DELETE` | `std::fs::remove_file()` |
| `_SEEK` | `Seek::seek()` |
| `_MALLOC` / `_MFREE` | `Vec::with_capacity()` / 自動解放（Drop） |
| `_SETBLOCK` | 不要（プロセスメモリ管理は不要） |
| `_GETENV` | `std::env::var()` |
| `_GETDATE` / `_GETTIM2` | `std::time::SystemTime` または `chrono` クレート |
| `_EXIT2` | `std::process::exit()` |

## X68000固有の処理でRust移植時に不要になるもの

1. **メモリブロック縮小** (`_SETBLOCK`): X68000ではプロセス起動時に空きメモリを全て確保するため縮小が必要。Rustでは不要。
2. **ファイルバッファ管理**: X68000では手動でバッファを確保・管理するが、RustのBufReader/BufWriterで代替。
3. **060turbo拡張メモリ** (`_MALLOC3`): X68000固有のメモリ拡張。Rust では不要。
4. **テンポラリファイル**: メモリが限られるX68000ではディスクのテンポラリファイルを使うが、Rustでは全てメモリ上に保持できる。

## エラーコード（主要なもの）

Human68kのシステムコールはエラー時に負の値を返します。

| 値 | 説明 |
|---|---|
| -1 | エラー（一般） |
| -2 | ファイルが見つからない |
| -3 | パスが見つからない |
| -4 | ファイルハンドルが足りない |
| -5 | アクセス拒否 |
| -6 | ファイルハンドルが不正 |

## IOCTRL（デバイス制御）

HASでは標準出力がターミナルかファイルかの判定に使用することがあります。
ただし実際のHASコードでは標準的なファイルI/Oのみ使用されています。

## 備考

- `$FF50`～`$FF7F` の番号は68030以上では使用不可（Memory Mapに重複）
- `$FF80`～は Human v2.15 以降で定義
- `$FF90`～`$FF93` は 060turbo.sys 独自拡張
- DOSコール発行方法: `TRAP #1` の後に機能番号ワードを置く

```68000asm
; 実例: ファイルオープン
move.w  #0, -(sp)    ; オープンモード（読み込み）
pea     filename     ; ファイル名
DOS     _OPEN        ; = .dc.w $FF3D
addq.l  #6, sp
tst.l   d0
bmi     error
```
